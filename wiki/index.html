NAME

fullsafe - create and manage backups and versions.

SYNOPSIS

fullsafe <command> [options] args

DESCRIPTION

fullsafe is a backup tool that helps to create and manage backups. It provides some sub-commands to deal with backups.
It stores all the backup files in the respective directories in the central location. This central location resides in home directory that can be found by name .fullsafe_backups.

COMMANDS

Command Description Options Syntax
init Initialise the fullsafe on the directory. - fullsafe init <directory files>
backup Create backup [-z zip archieve] fullsafe backup [-z]
restore Restore backup - fullsafe restore <backup file> <destination>
list List backups for the directory - fullsafe list
delete Delete backup - fullsafe delete <backup file>
content Show contents in the backup files - fullsafe content <backup file/s>
update Update fullsafe configurations

[-af add files]
[-rm remove files]
[-d rename backup directory]
[-v change version]

fullsafe update -af <directory files>
fullsafe update -rm <files>
fullsafe update -d <new directory name>
fullsafe update -v <new version>

--help Display fullsafe usages and commands - fullsafe --help
manual Open fullsafe manual in the browser - fullsafe manual
--version Display fullsafe version - fullsafe --version

SETUP & CONFIGURATIONS

Before creating backups, fullsafe needs to be initialized on the directory. Every initialisation is separate for different directories. Post initialisation, backup can be created.
fullsafe configurations are stored in the .fullsafe.config file in the directory. If this file is absent, fullsafe needs to be reinitialised.

EXAMPLES

Below are the some examples :

$ fullsafe init * $ fullsafe backup $ fullsafe restore 20211207_092102_v20.tar.gz myBackup $ fullsafe delete 20211207_092102_v20.tar.gz
Changing the backup directory name
$ fullsafe update -d newBackupDirName

EXIT STATUS

fullsafe returns following exit status :

  • 0 for success
  • 3 for some functional error
  • 4 if backup file or any dependency is missing
  • 5 if there is wrong input from the user
